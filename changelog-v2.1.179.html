Claude Design MCP Integration #

NewGradual Rollout

A built-in first-party MCP server called claude_design connects to Anthropic's Design API at https://api.anthropic.com/v1/design/mcp. When enabled, Claude can mock up UI screens directly in your session. A tip reads "Use Claude Design to mock up screens before you build."

Details
  • Controlled by feature flag tengu_omelette_whisk; can also be force-enabled via CLAUDE_CODE_ENABLE_DESIGN_MCP=true
  • Server name claude_design is treated as first-party — it auto-attaches login OAuth tokens and renders with a bare name (no URL prefix in the UI)
  • Setting CLAUDE_CODE_ENABLE_DESIGN_MCP=1 bypasses the feature flag for self-hosting or testing
◆
Design MCP server name (search "claude_design") and URL (search "https://api.anthropic.com/v1/design/mcp")

CLAUDE_CODE_ALTGR_AS_TEXT Env Var #

Env VarWindows

New env var for Windows users on keyboards with an AltGr key. AltGr is interpreted as Ctrl+Alt in Windows, which interferes with Claude Code shortcuts — this var forces AltGr combos to produce text instead.

Details
  • true/1: force mode — AltGr combos always produce text
  • false/0: off mode — AltGr combos pass through as Ctrl+Alt key events
  • Omitted: auto mode — enabled when WT_SESSION is present (Windows Terminal)
  • Only applies to printable character ranges (codepoints 32–126 and 160–55295)
◆
New env var handler (search "CLAUDE_CODE_ALTGR_AS_TEXT")

CLAUDE_CODE_OTEL_DIAG_STDERR Env Var #

Env VarTelemetry

When set, OpenTelemetry diagnostic errors are written directly to stderr in addition to the internal debug log — useful when debugging custom telemetry or OTEL integrations.

Details
  • Without this flag, OTEL errors only appear in the internal log (visible via /debug)
  • With the flag, errors stream to process.stderr prefixed with "[3P telemetry] OTEL diag error:"
  • The flag is stripped before spawning subprocesses so it doesn't leak
◆
New stderr path (search "CLAUDE_CODE_OTEL_DIAG_STDERR")

CLAUDE_CODE_SUPPRESS_SESSION_ATTRIBUTION Env Var #

  • Only applies in remote mode — no effect in interactive sessions

ExitWorktree: Graceful cwd Recovery #

Reliability

When exiting a git worktree, if the original working directory no longer exists, Claude now tries fallback locations in order: the original worktree path, the home directory, then the process root.

Details
  • Shows a clear message: "The original directory /path/to/old-dir no longer exists, so the session is now in /home/user."
  • Suggests restarting Claude from an existing directory, unless the fallback landed back in the worktree itself
◆

Git Log Isolated from Local Hooks and fsmonitor #

Reliability

When scanning recently-added files, Claude now passes four safety flags to git: core.fsmonitor=, core.hooksPath=/dev/null, core.pager=, and log.showSignature=false.

Details
  • Prevents fsmonitor daemons from slowing the call
  • Prevents local git hooks from running
  • Prevents pagers from blocking output
  • Prevents commit-signature verification from failing on machines without GPG
