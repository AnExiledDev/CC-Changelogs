claude-sonnet-5 added to model compatibility list #

New

The identifier "claude-sonnet-5" was added to an internal list of model names that gates a specific runtime behavior. The list already included claude-sonnet-4-0, claude-sonnet-4-5, claude-sonnet-4-6, and claude-haiku-4-5. Functions that check this list return false for any model on it, which typically means a legacy-mode fallback or optional feature is suppressed when using one of these newer models — a pattern the codebase uses to avoid applying compatibility shims to models that don't need them.

Users who configure Claude Code to use claude-sonnet-5 will now receive the same behavior as other recent Sonnet variants rather than triggering an unintended legacy code path.
