Version 2.1.204

This is a same-day patch release (built ~22 hours after 2.1.203) containing a single behavioral fix for the --output-format stream-json mode when combined with --verbose. Buffered output is now flushed at session startup in this mode, preventing messages from being silently dropped.

Bug Fixes

1 item
  • Stream-json verbose mode now flushes buffered output at startup. When running with --output-format stream-json and --verbose together, any messages queued before the session writer was ready are now delivered immediately on initialization, then the flush handler is cleaned up. Previously these early messages could be lost. Evidence: new startup flush function (uses outputFormat === "stream-json" and Boolean(l.verbose) check, search for "stream-json" near the added Fdf call at line ~884227)
