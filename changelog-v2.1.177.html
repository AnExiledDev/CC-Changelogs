Server-Side Model Error Overrides #

Infra

A new tengu-model-error-overrides config key allows Anthropic to customize the error messages and model-picker hint text users see when a model encounters an error — without shipping a client update.

Details
  • Delivered via the existing server-side feature-flag config mechanism; no user-facing setting
  • Maps model identifiers to objects with optional block (error message text) and pickerHint (hint shown in model picker) fields
  • When a matching override is present and its block string is non-empty, the override text replaces the default error display
  • If pickerHint is absent, the block text is reused as the picker hint
  • Any read error falls back to default behavior and logs model-error-overrides picker hint failed: or tengu-model-error-overrides block check failed:

Saffron Lattice Restricted to First-Party Builds #

Updates

The eligibility check for the saffron lattice (progressive-rollout auto-update feature) now requires the installation to be a first-party build. Previously this guard was absent, meaning third-party or forked builds could inadvertently enter the saffron lattice update path.

Saffron Lattice Default State Made Explicit #

Updates

The fallback value used when the tengu_saffron_lattice feature flag is absent or fails to parse changed from an empty object {} to an explicit { enabled: false }. If server config is unavailable, the feature is now definitively off rather than potentially indeterminate.
