/design-login #

NewAuth

A new slash command that authorizes design-system (claude.ai/design) access independently of the main session login — works even when Claude Code authenticates via API key or provider token.

Details
  • Opens a browser OAuth consent page specific to design-system scopes (user:design:read, user:design:write)
  • On SSH or remote sessions where a browser can't reach the local listener, falls back to a manual code-paste method
  • Credential stored as a separate designOauth entry in settings — entirely separate from the main session token; signing out of claude.ai does not invalidate it
  • Auto-refreshes; if refresh fails the command re-prompts. Requires CLAUDE_CODE_DESIGN_OAUTH_CLIENT_ID to be set
  • 5-minute browser-authorization timeout; after that the manual paste flow is suggested
◆
Design OAuth lock and flow (search "Design login", ".design_oauth_refresh.lock", "design login — approving opens a browser OAuth consent")

Agent tool: isolation: "remote" #

NewAgents

The Agent tool's isolation parameter now accepts "remote" in addition to "worktree", launching the subagent in a remote cloud (CCR) environment as a background task.

Details
  • Always runs in the background; you are notified when it completes
  • Remote agents run against the repository's default branch if the current local branch is not pushed to origin
  • If CCR is unavailable the tool reports [remote agent] isolation:'remote' is unavailable
  • Local agent context or no git root causes automatic fallback to other isolation modes
◆
Remote agent branch check (search "[remote agent] local branch '", "isolation:'remote' is unavailable", "CLAUDE_CODE_REMOTE_SESSION_ID")

/thinking: thinking_display parameter #

NewUX

The /thinking command now accepts an optional second argument to set the thinking display mode for the rest of the session.

Usage
  • /thinking 10000 summarized — show collapsed thinking summaries
  • /thinking 10000 omitted — hide thinking blocks entirely
  • /thinking 10000 — keeps the session-start --thinking-display value
  • /thinking 0 — disables extended thinking

Named-teammate swarms (TeamCreate/TeamDelete removed) #

BreakingAgents

The explicit TeamCreate and TeamDelete tools have been removed. Multi-agent coordination is now done entirely through the Agent tool's name parameter — no separate team-creation step needed.

Migration
  • Before: call TeamCreate, then Agent with team_name + name, then TeamDelete
  • After: just Agent({ name: "researcher", prompt: "..." })
  • The team_name field in TeammateIdle hook events is deprecated — will be removed in a future release
  • "main" is now a reserved recipient name for SendMessage, routing directly to the main conversation
◆
Removal of TeamCreate/TeamDelete (search "TeamCreate" absent from new file; "spawning named teammates with the"; "SendMessage routes it to the main conversation")

Code review: CLAUDE.md conventions angle #

Skills

The /code-review skill now runs a dedicated "conventions" finder angle that checks changed code against CLAUDE.md files governing those files.

Details
  • Inspects user-level ~/.claude/CLAUDE.md, repo-root CLAUDE.md, and any ancestor-directory CLAUDE.md / CLAUDE.local.md relevant to changed files
  • Only flags violations when the exact rule can be quoted and the exact violating line identified — no vague style inferences
  • High effort: 7 → 8 angles; max effort: 9 → 10 angles

Updated phase header (search "3 correctness angles + 3 cleanup angles + 1 altitude angle + 1 conventions angle")

Foundry deployment capability detection #

Azure

When running against an Azure AI Foundry deployment, Claude Code now detects unsupported capabilities at runtime and degrades gracefully instead of failing.

Details
  • Detects when Foundry returns a 400 error for: tool_search_server, tool_search, structured_outputs, web_search
  • On subsequent requests, unsupported capabilities are stripped (e.g. defer_loading and strict removed from tool definitions)
  • Surfaces user-friendly messages: "Web search is not available on this Foundry deployment." / "Tool search disabled: Foundry deployment for '…'"
◆
Search "[foundry-capabilities] deployment", "retry:foundry-capability-strip:", "not supported in your workspace"

Artifact tool: HTML skeleton wrapping at publish time #

BreakingArtifact

The Artifact tool now wraps content in a full <!doctype html>…<body> skeleton automatically at publish time. Files should be written without the outer HTML structure tags.

Migration
  • Write body content directly (CSS, scripts, page elements) — no <!DOCTYPE>, <html>, <head>, or <body> tags
  • Existing artifact files that include these tags may render with duplicate structure
  • Recommended location for artifact files: the scratchpad directory listed in the system prompt
◆
Updated Artifact tool description (search "wrapped in a", "<!doctype html>", "write the page content directly")

Skill directory scoping #

Skills

Skills can now be scoped to a specific directory subtree. When both a scoped and an unscoped variant of the same skill exist, the Skill tool picks the most-specific matching directory.

Details

  • When files are under a variant's directory, that variant wins (most specific directory wins)
  • Marketplace plugin entries can declare specific skill subdirectory paths relative to the plugin root
