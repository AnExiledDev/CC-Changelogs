Footer Link Badges (footerLinks) #

NewSetting

A new footerLinks setting lets you define regex patterns that are matched against each turn's output. When a pattern matches, the REPL footer shows a clickable badge linking to a URL built from named capture groups — useful for turning ticket IDs, PR numbers, or trace links into one-click navigation.

Details
  • Configured in user, flag, or managed settings only — not project .claude/settings.json
  • Entries use type: "regex" with pattern, url, and optional label fields
  • The url field is a URL template; {name} placeholders are filled from named regex capture groups (values are URL-encoded)
  • The URL origin must be a literal https:// or http:// scheme — dynamic origins are rejected
  • {name} in the label field is also filled from capture groups; defaults to the full match text
  • The existing PR link badge (detected from the current git repo) renders as the first footer badge; a dedicated URL template setting accepts {host}, {owner}, {repo}, {number}, {url} placeholders
◆
Search "[footerLinks] pattern" and "repl_footer_links"

Explicit Fork Agent Type #

NewAgents

subagent_type: "fork" is now a named, explicitly-selectable option in the Agent tool rather than the implicit default when no type is provided. This makes multi-agent orchestration code clearer and avoids accidental forking.

Details
  • Omitting subagent_type previously implicitly created a fork when the fork experiment was active; now forking requires an explicit "fork" value
  • Remote sessions cannot combine isolation: "remote" with fork — a clear error is returned
  • Other named agent types still start with zero context
◆

Project Tool: File Uploads Visible and Readable #

Projects

The Project tool now surfaces file uploads (PDFs, images, and other binary files) alongside text docs in project_info, project_read, and project_delete.

Details
  • project_info now includes a ## Project files (N) section for uploads and a ## Synced sources (N) section for connector-synced content (Google Drive, GitHub, Outline, MCP)
  • project_read on a file upload returns a file_kind field; document-type uploads (PDF, docx) return extracted text; other types return empty content with an explanatory notice
  • project_delete on a file upload returns a clear error explaining that uploads must be removed from claude.ai
  • The "not found" error now says "No doc or file at" and lists both docs and files

Grep Tool: Expanded rg Flags #

Tools

Five additional rg flags are now on the allowlist for the Grep tool.

New flags
  • --smart-case — case-insensitive if pattern is all-lowercase, case-sensitive otherwise
  • --case-sensitive — always case-sensitive (overrides --ignore-case)
  • --no-line-number — suppress line numbers
  • --multiline — allow patterns to match across newlines
  • --pcre2 — use PCRE2 regex engine
◆

rm/rmdir Safety Analysis Expanded #

Safety

The dangerous-removal checker that guards rm and rmdir has been significantly reworked with several new detection patterns.

New detections
  • Detects when a cd precedes a relative glob removal, making the target statically unresolvable
  • Detects when the removal target after glob expansion cannot be resolved to a directory
  • Detects when a glob traverses multiple directories that cannot be statically enumerated (more than one wildcard level deep)
  • Now checks against all configured working directories — additional working dirs and their parents are also protected
  • The old "remove CWD or its parent" check is replaced by the more precise "workspace directory" check
◆
Search "This command changes directories before the removal" and "This command's glob pattern traverses directories"

sed Command Static Analysis #

Safety

A new static analyzer for sed commands validates redirect targets in sed pipelines. Commands that carry redirect-borne content that cannot be statically validated, or that are over-length / contain ambiguous characters, now require explicit approval rather than being auto-allowed.

MCP First-Party Auth Error Handling #

MCP

When a first-party MCP server rejects the connection, Claude now emits a structured FIRST_PARTY_AUTH_REJECTED error with an actionable message — and automatically refreshes the token and retries once on a 401.
