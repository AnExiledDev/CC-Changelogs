Artifact Tool #

NewTool

Claude can now render an HTML or Markdown file to a private, shareable web page hosted on claude.ai. The resulting URL can be shared with teammates. Markdown is automatically converted to styled HTML.

Usage
pythonArtifact(file_path="report.html", favicon="📊")
Artifact(file_path="notes.md", favicon="📝", label="v2")
Details
  • Accepts .html, .htm, and .md files
  • favicon (required) — one or two emoji, no markup — sets the browser-tab icon
  • label (optional) — human-readable version label shown in the version picker
  • url (optional) — redeploy to an existing artifact URL the user provides
  • A strict CSP blocks all external resources — inline all CSS/JS and embed assets as data: URIs
  • Same file path → same URL on redeploy; different file path → new URL
  • Disable via CLAUDE_CODE_DISABLE_ARTIFACT env var or disableArtifact setting
  • Gated by feature flag tengu_cobalt_plinth; enabled for eligible claude.ai accounts
◆

--plugin-dir-no-mcp Flag #

NewFlag

Loads a plugin directory without also reading its .mcp.json file. Intended for SDK hosts that register MCP servers themselves rather than delegating to the engine.

Usage
bashclaude --plugin-dir-no-mcp <path>
Details
  • Behaves exactly like --plugin-dir except the engine skips .mcp.json / mcpServers discovery for that plugin
  • Equivalent setting: noMcp: true in the plugin frontmatter
◆
Search "Like --plugin-dir but the engine will not read this plugin's .mcp.json"

  • Unauthenticated: "ultra (cloud review) requires a claude.ai account — sign in to claude.ai to use it"
  • All messages include a link to https://code.claude.com/docs/en/ultrareview

web_fetch URL Provenance Check #

NewSecurity

web_fetch now enforces a provenance check — it can only retrieve URLs that appeared in a user message or a prior web_fetch result. Fetching URLs outside this set presents a permission prompt.

Details
  • If the user approves, the fetch proceeds; if denied or timed out, Claude receives an error
  • Timeout message: "The permission request for this URL was not answered in time. Ask the user to approve the fetch or include the URL in a message, then try again."
  • Error type: PROVENANCE_REQUIRED
  • Suppressed in bypassPermissions and dontAsk modes
◆
Search "URL not in provenance set. web_fetch can only retrieve URLs that appeared in a user message"
